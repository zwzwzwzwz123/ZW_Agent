# RAG + Agent 简历项目说明

这个示例知识库描述的是一个面向“大模型应用算法工程师”岗位的作品级项目。

项目目标是构建一个完整的检索增强生成系统，并加入透明的 Agent 工作流。它不应该只是普通的 PDF 聊天机器人，而应该展示检索决策、重排分数、引用证据、失败模式和可调参数。

核心模块包括：文档加载、文本切分、Embedding、向量检索、关键词检索、混合召回、重排、答案生成和 Web UI。UI 中应该展示足够多的中间过程，方便面试时解释系统如何工作。

混合检索结合了语义向量检索和 BM25 关键词检索。向量检索适合处理同义改写和语义相近的问题；BM25 适合处理精确术语、缩写、实体名、接口名和公式名。

chunk 大小是一个重要取舍。较小的 chunk 能提高定位精度，但可能丢失上下文；较大的 chunk 能保留更多上下文，但容易引入噪声并浪费 prompt token。初始可以选择 500 到 900 左右的大小，并设置 10% 到 20% 的 overlap。

重排模块的作用是把第一阶段召回的候选文本块重新排序。第一阶段召回更关注覆盖率，第二阶段重排更关注相关性、证据质量和多样性。本项目默认使用 MMR 风格的轻量重排，也支持切换到 Cross-Encoder 精排。

Agent 工具编排适合处理需要多步决策的问题。在这个项目中，Agent 会分析问题意图、检查知识库、执行混合检索、调用重排、估计证据置信度，并生成带引用的回答。

好的面试讲解点包括：检索评估、幻觉控制、chunk 策略、Embedding 模型选择、query rewrite、rerank、可观测性、缓存、延迟和成本权衡。

常见失败模式包括：知识库内容过期、文档缺失、问题歧义、chunk 质量差、重复 chunk、引用不准确、证据不足但回答过度自信。生产系统需要评估集、反馈闭环和线上监控。

项目后续可以扩展多查询召回、Cross-Encoder 重排、LangGraph 工作流、工具调用、文档权限过滤、会话记忆，以及 recall@k、MRR 等离线检索指标。
