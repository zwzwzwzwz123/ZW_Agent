# 大模型应用算法工程师面试知识

大模型应用算法工程师需要同时理解模型行为和应用系统架构。岗位通常要求候选人具备 prompt 设计、RAG、Agent、评估、优化和部署意识。

RAG 系统的价值在于解决大模型无法可靠掌握私有知识、实时知识和领域知识的问题。检索模块在推理时为模型提供外部证据，使回答能够被知识库约束。

基础 RAG 流程包括：加载文档、切分文档、为文本块生成 Embedding、写入向量库、根据用户问题召回候选文本块、可选重排、最后基于检索上下文生成回答。

重要算法选择包括：chunk 粒度、Embedding 模型、索引类型、top-k、混合检索权重、rerank 方法、prompt 模板和置信度估计。

优秀的 RAG 回答应该带引用。引用让系统可审计，也能降低幻觉风险。如果证据不足，系统应该明确说明缺少信息，而不是编造答案。

Agent 是能够根据状态选择动作的系统。在大模型应用中，Agent 可以调用工具、检索文档、执行计算、提出澄清问题，或者规划多步任务。

Agent 常见失败包括：过度规划、调用不必要的工具、循环执行、相信弱证据、缺少终止条件。生产级 Agent 需要工具 schema、步数限制、错误恢复、权限控制和可观测性。

评估是 RAG 项目的核心。常见指标包括检索 recall@k、MRR、答案忠实度、上下文精确率、引用准确率、延迟和成本。项目早期可以先构建小规模人工标注评估集。

优化应该从度量开始。常见优化手段包括文档清洗、chunk 策略调整、metadata filter、混合检索、query expansion、rerank、prompt 约束、缓存和模型选择。

对于简历项目，最强信号不只是应用能跑，而是候选人能解释每个设计选择为什么存在，以及在真实业务约束下如何继续优化。
